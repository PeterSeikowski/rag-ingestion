## Step 12b — Architecture-compliance sweep and code review
**Status:** Done
**What happened:** Ran a multi-agent workflow (6 parallel compliance checks against the 14 hard rules + 3 layer-scoped code reviews, then an independent adversarial verification pass over every candidate finding). Raw output: 22 candidate findings, 21 CONFIRMED, 1 REFUTED (a low-value "no negative-space test" nitpick). All 21 confirmed findings were triaged and every one was either fixed or explicitly documented as a known limitation — none were dropped silently.

**Fixed (real bugs and gaps):**
- **[high] ChunkRecord ACL/metadata field loss on the PDF path** — `Chunker.chunk()` has no way to receive tenant/ACL/metadata, so every PDF-derived `ChunkRecord` was silently missing `tenant_id`/`allowed_groups`/`classification`/`custom_metadata` even though `DocumentRecord` got them correctly and the README explicitly promised chunk-level ACL support. Fixed by back-filling these fields onto every chunk in `run_pdf_ingestion` right after chunking.
- **[high] `window_text()` was O(n²)** — re-decoded the token prefix from index 0 on every window boundary; benchmarked at 30+ seconds for a ~250-page-equivalent document, reachable via the public `chunking_config` API field with no upper bound relating `chunk_overlap_tokens` to `chunk_size_tokens`. Fixed with an O(n) cumulative-offset precomputation (`_cumulative_char_offsets`, decodes each token exactly once) — benchmarked at ~1s for a 250k-token document (was 30s+). Also added a `ChunkingConfig` validator rejecting `chunk_overlap_tokens >= chunk_size_tokens` outright (was previously silently accepted and degraded to a near-duplicate window per token).
- **[high] `document_id` regenerated on every Celery retry** when no `external_id` was supplied, breaking the delete-then-insert idempotency the retry logic depends on — a retry would mint a fresh random ID, so `delete_document()` on retry would never find the first attempt's partial writes, leaving orphaned/duplicate data. Fixed: `_resolve_document_id` now reuses the job's already-stored `document_id` if one was set by a prior attempt.
- **[high] Indexing failures were blanket-classified as `TransientIngestionError`**, so a deterministic bulk-write failure (e.g. a mapping conflict) would be retried 3 times pointlessly after `delete_document` had already removed the document's previously-good records, permanently losing them. Fixed: `_embed_and_index` now classifies a plain `RuntimeError` (what `ElasticsearchVectorStoreWriter` raises for a deterministic bulk-item failure) as `PermanentIngestionError`, keeping only genuine connectivity-style exceptions as transient/retryable.
- **[medium] Empty-text HEADING/TITLE elements corrupted `section_path`** in `docling_pdf_parser.py` — the heading-stack pop ran unconditionally while the push was gated on non-empty text, so a heading with missing text would permanently truncate the section hierarchy for every following element. Fixed: pop and push now happen together, only when there's real heading text.
- **[medium] A job could get stuck at `PENDING` forever** if Celery enqueue itself failed (e.g. broker briefly unreachable) — the exception was re-raised to the caller (500) but the already-created job record was never marked `FAILED`. Fixed: both `IngestionService.submit_pdf_ingestion`/`submit_metadata_ingestion` now call `job_service.mark_failed()` before re-raising.
- **[medium] `Dockerfile` shipped `pytest`/`pytest-mock`** (and transitive deps) in the production runtime image. Fixed: pruned via `pip uninstall` in the builder stage before the venv is copied into the runtime stage.
- **[low] `Dockerfile`'s runtime stage had no system libraries** for docling's torch/opencv dependency chain (`libgomp1`, `libglib2.0-0`) — would surface as an `ImportError` the first time a real PDF job runs, not at build time (docling is imported lazily). Fixed: added `apt-get install libgomp1 libglib2.0-0 libgl1` to the runtime stage.
- **[low] Duplicated `page_start`/`page_end` min/max logic** in both chunkers' table-chunk construction, bypassing the existing `aggregate_source_refs()` helper both files already use elsewhere. Fixed: both now call `aggregate_source_refs([element])`.
- **[low] Temp per-job directory (`TEMP_UPLOAD_DIR/{job_id}/`) was never removed**, only the file inside it — leaking one empty directory per PDF job forever. Fixed: `_remove_temp_file` in `workers/tasks.py` now also `rmdir`s the (now-empty) parent directory.
- **[low] `ChunkingConfig`'s docstring inaccurately claimed** its hardcoded 512/64 defaults are used when a caller omits `chunking_config` entirely — they're actually only reached on a *partial* client override, since `ingestion_pipeline.py` always builds an explicit config from `Settings` for the omitted-entirely case. Docstring corrected.
- **[low] PDF uploads were fully buffered into memory before the size check ran** (`await file.read()` with no bound) — `MAX_UPLOAD_SIZE_BYTES` didn't actually bound memory usage for an oversized upload. Fixed: `_read_upload_within_limit` now reads in 1 MiB chunks and rejects as soon as the limit is crossed.
- **[test-coverage, several]** Added `tests/unit/test_workers_tasks.py` (12 tests: `_IngestionTask.on_success`/`on_failure`, temp-file+directory cleanup, task-body delegation, `PermanentIngestionError` propagation, autoretry configuration — previously zero coverage on the entire Celery task layer), `tests/unit/test_job_service.py` (11 tests covering every `JobService` transition including `mark_failed`, previously never invoked by any test), `tests/unit/test_architecture_boundaries.py` (7 tests: `application/`'s own infra-import boundary, single-adapter-only libraries each importable from exactly one designated file, `fastapi`/`celery` scoped to their layers — previously only `domain/` had this kind of static enforcement), 3 new `GET /health/ready` tests in `test_api_routes.py` (200/503/container-build-failure paths — previously zero coverage), and regression tests in `test_ingestion_pipeline.py` for all three `ingestion_pipeline.py` bug fixes above (chunk ACL back-fill, document_id retry stability incl. a full flaky-then-retry scenario, indexing failure classification). Also strengthened two trivially-true `assert x is not None` assertions in `test_bootstrap.py` into real attribute checks.
